# 🤝 Guía de contribución

**¡Gracias por querer contribuir a este proyecto open source!**

Tu aporte es clave para mantener este repositorio activo, funcional y útil para toda la comunidad. A continuación, te explicamos cómo puedes colaborar y bajo qué condiciones.

---

## 🚀 Cómo contribuir

1. **Haz un fork del repositorio**
   - Crea una copia del repositorio en tu propia cuenta de GitHub.

2. **Crea una nueva rama**
   - Usa un nombre descriptivo basado en la rama `main`.
   - Ejemplo:
     ```bash
     git checkout -b mejora-docs
     ```

3. **Realiza tus cambios**
   - Puedes:
     - Añadir nuevas funcionalidades.
     - Corregir errores.
     - Mejorar la documentación.
     - Sugerir refactorizaciones o mejoras de rendimiento.

4. **Haz commit y push**
   - Describe claramente tus cambios:
     ```bash
     git add .
     git commit -m "Corrige validación en formulario de registro"
     git push origin mejora-docs
     ```

5. **Envía una Pull Request**
   - Explica los cambios realizados de forma clara.
   - Si estás solucionando un issue, menciona el número (#).

---

## 🔍 Proceso de revisión

- Un revisor analizará tu Pull Request y podrá hacer comentarios o sugerencias.
- Podrás modificar tu rama y los cambios se actualizarán automáticamente en el PR.
- Una vez aprobado, se fusionará a la rama principal.

---

## 🛡️ Reglas del repositorio

- ❌ **No se permite hacer push directo a la rama `main`**.
- ✅ **Todas las contribuciones deben hacerse mediante Pull Requests**.
- ✅ **Tu código debe estar limpio, ordenado y seguir el estilo del proyecto**.
- 🚫 **No se aceptan cambios sin justificación o sin utilidad clara**.

---

## 🐞 ¿Cómo reportar un problema?

Si encuentras errores, problemas o tienes sugerencias, abre un nuevo [issue](https://github.com/Darling-P11/imagrov2/issues) y describe:

- Qué ocurrió
- Cómo reproducirlo
- Qué comportamiento esperabas

---

## 🙌 Otras formas de contribuir

- Ayuda con la documentación o tutoriales.
- Sugiere ideas nuevas.
- Difunde el proyecto en redes o eventos.
- Reporta errores y problemas de usabilidad.

---

## 🎉 Gracias

Cada contribución cuenta. Ya sea pequeña o grande, tu ayuda mejora este proyecto y fortalece nuestra comunidad.
